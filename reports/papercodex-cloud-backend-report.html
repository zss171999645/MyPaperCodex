Paper Codex / Cloud Backend Briefing

云服务器可行，但要按产品后端来做。

结论不是“买一台机器跑接口”，而是把账号、同步、arXiv 信息流、公共论文解析、缓存和运维拆成清楚的生产边界。

FINAL VERDICT

可以买云服务器。第一台机器适合做生产起点；私有 PDF 不上云，公共 arXiv 解析可以上云但必须排队、限速、内网运行。

产品边界

商业化第一原则：云端同步的是用户元数据，不是用户的私有论文文件。

SYNC 云端同步

账号、设备、arXiv ID、保存状态、tags、folders、用户总结、用户笔记、收藏夹、黑白名单。

LOCAL 永远本地

用户私有 PDF、非 arXiv 全文、PDF 路径、页面文本、span、bbox、Codex workspace。

PUBLIC 可处理公共 arXiv

metadata、缩略图、标签、相似度、公开 PDF 的解析结果。App 仍优先链接回 arXiv 下载。

BILLING 收费预留

从第一版就有 plan、subscription、entitlement、usage counter，先不开收费也不阻碍以后商业化。

推荐架构

App 只连 Product API；CodeArXiv 和 parser 都退到内网服务。

macOS App
本地 PDF 阅读 本地索引 Keychain token 离线缓存 同步 outbox
Cloud Product Backend
Product API

注册、登录、同步、权限、计划、用户数据。

PostgreSQL + Redis

产品数据、任务队列、限流、状态。

CodeArXiv Worker

每日 feed、相似度、黑白名单、收藏夹。

Parser Worker

只解析公共 arXiv；GROBID 内网，不公开。

论文解析策略

解析不是一个开关。不同论文来源要走不同链路。

非 arXiv 私有 PDF

用户手动导入的文件。

不上云

不上传文件、不传 hash、不传全文、不传本地路径。

本地解析

PDFKit / 本地 index / 本地 reader / 本地 Codex context。

公共 arXiv PDF

公开论文，有 arXiv ID。

可上云处理

metadata、缩略图、相似度、公开结构化信息。

队列解析

worker 拉任务，GROBID 内网解析，失败可重试。

PDF 分发

用户点 Open / Download。

不做 PDF 中心

避免版权和带宽风险。

链接回 arXiv

App 根据 arXiv URL 下载，保存到本地或缓存。

服务器规格

第一台服务器能跑起来，但要知道它的边界。

MINIMUM
2C / 4G
只适合轻量内测

API、PostgreSQL、Redis、少量 worker。不要在这里跑批量 PDF 解析。

RECOMMENDED
4C / 8G
生产起点

账号、同步、Discover、缩略图、轻量公共 arXiv 任务。适合早期真实用户。

PARSER
4C / 16G
带 GROBID 解析

公共 arXiv 全文结构化。解析服务内网部署，并发限制在 1-2 起步。

生产演进路线

不是一步到位，但每一步都能平滑升级。

PHASE 1
单机生产起点

Docker Compose 跑 API、DB、Redis、worker、CodeArXiv、Caddy。

PHASE 2
账号和同步

邮箱注册、验证、登录、Keychain、metadata sync、设备管理。

PHASE 3
公共 arXiv 后台任务

feed、缩略图、相似度、标签、缓存状态、解析队列。

PHASE 4
拆服务

PostgreSQL/Redis 托管化，parser/worker 独立机器，CDN 缓存静态资产。

PHASE 5
商业化

entitlement 生效、付费计划、使用量统计、管理员支持、数据导出删除。

Purchase Decision
可买

建议先买 4C / 8G / 100GB SSD / Ubuntu 24.04。若确定要在同机跑 GROBID，直接买 4C / 16G / 200GB SSD。

1
给 SSH，不给云厂商主账号 最好用 SSH key；临时密码也可以，但配置后会关掉密码登录。
2
域名指向 API 例如 api.papercodex.xxx，Caddy 自动 HTTPS。
3
邮件服务稍后配置 用于注册验证和找回密码；我会到那一步再告诉你填什么。
4
不要把 parser 暴露公网 GROBID / Redis / PostgreSQL 全部内网，只让 Product API 对外。

最终判断

这是一条能从内测走到收费产品的路线。

FEASIBLE 云服务器迁移可行

先单机跑产品 API 和后台 worker，成本低、控制强、能快速迭代。

PRIVATE 隐私边界清楚

私有 PDF 本地处理，云端只同步用户元数据，这是商业化卖点。

CONTROL 解析要限速排队

公共 arXiv 可以解析，但不要同步阻塞用户操作，也不要无节制爬取。

SCALABLE 后续可拆分

DB、Redis、worker、parser、CDN 都能从单机平滑迁出。
