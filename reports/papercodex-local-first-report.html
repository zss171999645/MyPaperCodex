Paper Codex / Local-First Briefing

先做一个真正好用的本地读论文工具。

arXiv 是入口，不是后端。CodeArXiv 是参考实现，不再是运行依赖。用户数据、PDF、标签、总结、笔记、缓存和 embedding 都放在本地。

FINAL VERDICT

当前版本不要账号、不要同步、不要云后端。先把 Library + Reader + 本地 Codex 处理 + arXiv 直连做扎实。

产品边界

这次收窄后，产品复杂度明显下降，但读论文主体验会更清楚。

LOCAL 所有用户数据本地

PDF、library、tag、folder、笔记、用户总结、Codex 结果、缩略图和缓存都在本机。

ARXIV Discover 直连 arXiv

从 arXiv 列表和 Atom API 拉 metadata，按日期和分类缓存，不再连 NAS feed。

OPTIONAL embedding 可选

Settings 填 base URL、key、model。没配置也能完整阅读，只是不做相似度排序。

NO CLOUD 暂不做后端

账号、同步、收费、云解析都从当前实现主线移除，保留为未来可能性。

核心流程

Discover 只负责找到论文；Library 和 Reader 才是主场。

01 拉 arXiv

按 category/date 获取 ID，再批量拉 Atom metadata。

02 本地缓存

保存 metadata、小图、可选 PDF，断网时可以继续浏览缓存日期。

03 Open

PDF 下载到可删缓存，显示进度，然后进入 reader/chat。

04 Codex 处理

本地生成总结、中文摘要和标签，结果写入本地数据库。

05 Save

用户选 tag，PDF 移到 library，之后离线可读可问。

设置界面要变成这些

删除 CodeArXiv 连接设置，改成本地能力配置。

保留和强化

Storage

library 路径、cache 路径、清理缓存、可选每日 PDF 预缓存。

arXiv

分类、日期刷新、缓存状态、直接 arXiv 网络错误提示。

Codex

自动总结、自动打 tag、重新生成、prompt 版本。

Prompts

快捷 prompt 的标题和内容，继续用于 reader chat 下拉。

新加和移除

Embedding

base URL、API key、model、启用开关、测试 provider。

Ranking

白名单 tag、黑名单 tag、相似度来源 folder/tag。

Remove

CodeArXiv base URL、username、token、favorite sync。

Later

账号、同步、收费、云解析只作为未来可选模块。

最终判断

这条路更贴近“读论文”本身，也更容易做成高质量桌面产品。

FIRST Library 优先

读、问、总结、做笔记，都围绕本地 library 走。

SIMPLE arXiv 只是入口

直接拉、直接缓存、直接下载，不维护自己的 feed 后端。

ROBUST 离线也成立

没有网络、没有 embedding provider、没有云账号时，核心阅读功能仍然可用。
